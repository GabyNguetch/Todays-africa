# 🔧 Résolution du problème 404

## Problème identifié
Les routes dynamiques (`/article/[id]` et `/category/[slug]`) retournaient une erreur 404 en production sur Vercel.

## Cause
L'`apiUrl` était configuré avec un chemin relatif `"api/proxy"` qui ne fonctionnait pas en production car :
1. Le proxy Next.js n'était pas correctement configuré pour la production
2. Les appels API échouaient silencieusement
3. Les pages dynamiques ne pouvaient pas récupérer les données

## Solution appliquée

### 1. Modification de `lib/constant.ts`
```typescript
apiUrl: process.env.NEXT_PUBLIC_API_URL || "https://totayafrica.onrender.com/api/v1"
```

### 2. Création des fichiers d'environnement
- `.env.local` : Pour le développement local
- `.env.production` : Pour le déploiement Vercel

### 3. Configuration Vercel
Dans les paramètres de votre projet Vercel, ajoutez la variable d'environnement :
```
NEXT_PUBLIC_API_URL=https://totayafrica.onrender.com/api/v1
```

## Déploiement

### Option 1 : Via Git (Recommandé)
```bash
git add .
git commit -m "fix: correction de l'apiUrl pour résoudre les 404 en production"
git push origin main
```
Vercel redéploiera automatiquement.

### Option 2 : Via Vercel CLI
```bash
npm run build
vercel --prod
```

- Le backend doit être accessible depuis `https://totayafrica.onrender.com/api/v1`
- Les CORS doivent être configurés sur le backend pour accepter les requêtes depuis `todays-africa.vercel.app`
- Les routes dynamiques nécessitent que le serveur Next.js soit actif (mode SSR)
